## ğŸ’¡ Key Concepts

### Application Credentials (You Manage)
- **API Key**: `BnTtll7mOpDttM0Ppa8LgHyr0RH5jYyACV2GxD4x0`
- **Private Key**: Your ECDSA P-256 private key
- **Base URL**: `https://api.liam.netxd.com`

### User Credentials (API Generates)
- **User Key**: Auto-generated by `/api/auth/create-profile`
- **Example**: `c0312641aade88080be0c7f0801bc4dc6edbeb2752cb1530ae0483728e6a4f81`

## ğŸ¬ Quick Example

```javascript
// 1. Your application credentials (one-time setup)
const API_KEY = 'BnTtll7mOpDttM0Ppa8LgHyr0RH5jYyACV2GxD4x0';
const PRIVATE_KEY = '...your ECDSA private key...';

// 2. Create user (auto-generates userkey)
const userKey = await createUserProfile(
  'John Doe',
  'unique_user_123',
  API_KEY,
  PRIVATE_KEY
);
// Returns: "c0312641aade88080be0c7f0801bc4dc6edbeb2752cb1530ae0483728e6a4f81"

// 3. Use userkey for all operations
await createMemory(userKey, 'I love pizza', 'preferences');
const memories = await listMemories(userKey, 'food');
```

## ğŸ” Authentication

All requests require:

```http
POST https://api.liam.netxd.com/api/endpoint
Content-Type: application/json
apiKey: <your-api-key>
signature: <ecdsa-signature-base64>
```

**Signature Algorithm**: ECDSA SHA256 with P-256 curve, DER-encoded, Base64

## ğŸ”’ Security Best Practices

1. **Never** expose private keys in client-side code
2. **Always** store credentials in environment variables
3. **Never** commit keys to version control
4. **Use** HTTPS for all API calls (default)
5. **Validate** user input before sending to API
6. **Rotate** keys periodically in production
